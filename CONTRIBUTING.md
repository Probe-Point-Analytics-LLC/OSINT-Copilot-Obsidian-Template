# Contributing to OSINT Copilot Template

Thank you for your interest in contributing to the OSINT Copilot Obsidian Template!

## How to Contribute

### Reporting Issues

If you encounter issues with this template:

1. **Template-specific issues**: Open an issue in this repository
2. **Plugin issues**: Report them at the [OSINT Copilot Plugin repository](https://github.com/Probe-Point-Analytics-LLC/Obsidian-OSINT-Copilot-plugin)

### Suggesting Improvements

We welcome suggestions for:
- Additional example workflows
- Better folder organization
- Documentation improvements
- Setup automation

Please open an issue describing your suggestion.

### Submitting Changes

1. Fork this repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. **Important**: Ensure no sensitive data is included:
   - Remove any API keys
   - Clear personal investigation data
   - Sanitize example reports if needed
5. Test your changes with a fresh Obsidian vault
6. Commit your changes (`git commit -am 'Add new feature'`)
7. Push to the branch (`git push origin feature/your-feature`)
8. Open a Pull Request

## Security Guidelines

### Never Commit Sensitive Data

- **API Keys**: Never commit `data.json` files containing license keys
- **Investigation Data**: Be careful with example reports and entities
- **Personal Information**: Remove any PII from examples

### Before Submitting

Run this checklist:
- [ ] No API keys in any files
- [ ] Example data is sanitized or fictional
- [ ] `.gitignore` properly excludes sensitive files
- [ ] Documentation is clear and professional

## Code of Conduct

- Be respectful and professional
- Focus on constructive feedback
- Help maintain a welcoming community

## Questions?

For questions about:
- **This template**: Open an issue here
- **The plugin**: Visit the [plugin repository](https://github.com/Probe-Point-Analytics-LLC/Obsidian-OSINT-Copilot-plugin)
